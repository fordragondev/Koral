# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source decisions:** All values marked [CONTEXT] are locked by CONTEXT.md decisions.
> Values marked [DEFAULT] are applied from standard conventions where upstream artifacts were silent.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | none — initialize with `pnpm dlx shadcn@latest init` before executor begins |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | System font stack — Inter via `next/font/google` (Inter is the shadcn default; apply from Phase 1 so all phases inherit the same base) |

**shadcn components required in Phase 1:**

| Component | Usage |
|-----------|-------|
| `button` | Primary CTA, OAuth buttons, sign-out |
| `input` | Email field on sign-in form |
| `form` | react-hook-form wrapper |
| `label` | Form field labels |
| `separator` | "or continue with" divider between OTP and OAuth |
| `toast` | Error feedback (invalid code, rate limit, etc.) |
| `skeleton` | Loading states during tenant resolution and auth |

All components are installed via `npx shadcn add {component}` from the official shadcn registry only. No third-party registries in Phase 1.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, gap between OTP digit cells, inline padding |
| sm | 8px | Gap between form label and input, button icon padding |
| md | 16px | Default form field spacing, card body padding, nav item gap |
| lg | 24px | Card padding, section spacing within auth form |
| xl | 32px | Auth card vertical margin from center, page-level padding |
| 2xl | 48px | Major section breaks (nav to content area) |
| 3xl | 64px | Page-level top/bottom spacing on auth page |

Exceptions:
- OTP digit cell: minimum 44px height (touch target requirement — WCAG 2.5.5). Width: 44px. [DEFAULT: accessibility]
- Nav bar: 64px height on desktop, 56px on mobile. [DEFAULT: standard nav height]
- Language toggle and theme toggle touch targets: minimum 44px x 44px. [DEFAULT: accessibility]

## Typography

Font family: Inter (via `next/font/google`, subset: latin, latin-ext for ES diacritics).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (Regular) | 1.5 | Form helper text, nav labels, general copy |
| Label | 14px | 500 (Medium) | 1.4 | Form field labels, button text, toggle labels (EN / ES) |
| Heading | 24px | 600 (Semibold) | 1.25 | Auth card title ("Welcome to Aquarium Community") |
| Display | 36px | 700 (Bold) | 1.2 | Reserved — not used in Phase 1 UI (no hero content shipped in Phase 1) |

Declared weights: 400 Regular, 500 Medium, 600 Semibold, 700 Bold.

> Note: 500 Medium is required for button and label legibility at 14px. This is the only phase where 4 weights are declared; subsequent phases should consolidate to 2 if possible. The split is justified because the auth UI is the first impression of the platform.

## Color

### Aquarium Vertical — Deep Ocean Theme

Two intentional modes: dark (deep water) and light (bright reef). [Source: CONTEXT.md D-21, D-27–D-31]

Colors are defined as CSS custom properties on `:root[data-theme="dark"]` and `:root[data-theme="light"]` in `globals.css`. All Tailwind classes reference these tokens. Per-vertical overrides come from the `tenants` config row — aquariumcommu.com tokens are the defaults.

#### Dark Mode (Deep Water)

| Role | CSS Token | Hex Value | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `--color-bg` | `#0a1628` | Page background, auth page background |
| Secondary (30%) | `--color-surface` | `#0f2240` | Auth card background, nav bar background |
| Accent (10%) | `--color-accent` | `#00bcd4` | Primary button fill, active OTP digit border, focused input ring, language/theme toggle active indicator |
| Accent hover | `--color-accent-hover` | `#00acc1` | Button hover state |
| Destructive | `--color-destructive` | `#ef4444` | Error toasts, invalid code state, rate limit warning |
| Text primary | `--color-text` | `#e2f0f7` | Body text, headings, labels |
| Text muted | `--color-text-muted` | `#7ba3bb` | Helper text, placeholder text, disabled state |
| Border | `--color-border` | `#1e3a5f` | Card borders, input borders, OTP cell borders, dividers |

#### Light Mode (Bright Reef)

| Role | CSS Token | Hex Value | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `--color-bg` | `#f0f8ff` | Page background |
| Secondary (30%) | `--color-surface` | `#ffffff` | Auth card background, nav bar background |
| Accent (10%) | `--color-accent` | `#0097a7` | Primary button fill, active OTP digit border, focused input ring, language/theme toggle active indicator |
| Accent hover | `--color-accent-hover` | `#00838f` | Button hover state |
| Destructive | `--color-destructive` | `#dc2626` | Error toasts, invalid code state, rate limit warning |
| Text primary | `--color-text` | `#0a1628` | Body text, headings, labels |
| Text muted | `--color-text-muted` | `#4a6fa5` | Helper text, placeholder text, disabled state |
| Border | `--color-border` | `#b0cfe0` | Card borders, input borders, OTP cell borders, dividers |

Accent reserved for: primary CTA button fill, OTP digit cell active/focused border, text input focus ring, active state indicator on language toggle (EN|ES), active state indicator on theme toggle (sun/moon). Never used for decorative elements, hover backgrounds, or secondary text.

## Interaction Contracts

### Sign-In / Auth Form

The sign-in page is a single unified flow — no separate register page. [Source: CONTEXT.md D-05]

**Step 1 — Email entry:**
- Single email input centered on page
- Primary button: "Continue with email"
- "or" separator (Separator component, text: "or")
- OAuth buttons below: "Continue with Google" and "Continue with Apple"
- Helper text below email input: "We'll send you a 6-digit code. No password needed."

**Step 2 — OTP code entry:** [Source: CONTEXT.md D-08]
- 6 individual single-character input cells rendered side by side with xs (4px) gap between cells
- Each cell: 44px x 44px, border-radius 8px, border uses `--color-border`, font size 24px weight 600
- Behavior: typing a digit auto-advances focus to next cell; backspace clears current cell and moves focus to previous cell; paste of a 6-digit string fills all cells; on last digit entry, form auto-submits — no submit button shown
- Active/focused cell: border color switches to `--color-accent`, 2px ring
- Invalid/error state: all 6 cells border color switches to `--color-destructive`; cells shake (CSS keyframe, horizontal 4px offset, 300ms duration); error message appears below cells
- Loading state (after auto-submit): cells are disabled, show a subtle pulse animation on the primary button

**OAuth buttons:**
- "Continue with Google" — white/surface background, Google brand logo (SVG inline, 20px), label text 14px medium
- "Continue with Apple" — inverse (dark fill in dark mode, black in light mode), Apple logo (SVG inline, 20px), label text 14px medium
- Both buttons: full width, 44px height, border-radius 8px

### Language Toggle (EN | ES)

[Source: CONTEXT.md D-13, D-29]

- Always visible in nav bar
- Text format: `EN | ES` — plain text, no flag icons, no dropdown
- Active locale: `--color-accent` color, font weight 600
- Inactive locale: `--color-text-muted`, font weight 400
- The `|` separator: `--color-text-muted`, not interactive
- Touch target: 44px height minimum; the two labels and separator are wrapped in a single nav element
- On click: locale switches, URL prefix updates, no page reload if using next-intl client-side navigation

### Theme Toggle (Dark / Light)

[Source: CONTEXT.md D-27–D-31]

- Always visible in nav bar, placed immediately after EN|ES toggle
- Icon: `Moon` (dark mode active) or `Sun` (light mode active) from lucide-react, 20px
- Touch target: 44px x 44px minimum
- No label text — icon only
- On click: toggles `data-theme` on `<html>`, writes preference to localStorage immediately
- Transition: CSS `transition: background-color 200ms ease, color 200ms ease` on `:root` — smooth but not distracting

### Nav Bar

- Height: 64px desktop / 56px mobile
- Background: `--color-surface`
- Bottom border: 1px solid `--color-border`
- Left: tenant logo (SVG or next/image, max-height 32px)
- Right: EN|ES toggle, theme toggle, sign-in button (logged-out) or user avatar (logged-in)
- Sign-in button: accent fill, 14px medium, "Sign in" copy

### Loading / Skeleton States

[Source: CONTEXT.md — Claude's Discretion: "Loading/skeleton states during tenant resolution"]

- Tenant resolution happens in Edge Middleware — should be sub-ms; no skeleton required for this step
- Auth form initial load: no skeleton needed (static form)
- OTP submission loading: primary button shows spinner (lucide `Loader2` icon, 16px, spin animation) and is disabled; button text hidden
- Page-level loading (Suspense boundary): use shadcn `Skeleton` at 16px height for text lines, full-width for blocks, border-radius 4px

| Action | Confirmation approach |
|--------|----------------------|

## i18n Contract

All copy above must be authored in both English and Spanish. Spanish is a native language in the platform, not a translation layer. [Source: CONTEXT.md D-17]

Message file structure:

```
messages/
  en/
    common.json    # shared: errors, loading states
    nav.json       # nav bar labels, sign-in, sign-out, toggles
    auth.json      # auth flow: email entry, OTP, OAuth
  es/
    common.json
    nav.json
    auth.json
```

No string may be hardcoded in a component. Every user-facing string goes through `useTranslations()` or `getTranslations()`.

## Per-Vertical Theming Architecture

[Source: CONTEXT.md D-18–D-20, D-31]

The `tenants` table row for each vertical stores a `branding` JSONB column with the following shape (aquariumcommu.com defaults shown):

```json
{
  "name": "Aquarium Community",
  "tagline": "The home for aquarium hobbyists",
  "logoUrl": "/brands/aquarium/logo.svg",
  "faviconUrl": "/brands/aquarium/favicon.ico",
  "defaultLocale": "en",
  "dark": {
    "bg": "#0a1628",
    "surface": "#0f2240",
    "accent": "#00bcd4",
    "accentHover": "#00acc1",
    "destructive": "#ef4444",
    "text": "#e2f0f7",
    "textMuted": "#7ba3bb",
    "border": "#1e3a5f"
  },
  "light": {
    "bg": "#f0f8ff",
    "surface": "#ffffff",
    "accent": "#0097a7",
    "accentHover": "#00838f",
    "destructive": "#dc2626",
    "text": "#0a1628",
    "textMuted": "#4a6fa5",
    "border": "#b0cfe0"
  }
}
```

The root layout reads this JSONB, emits a `<style>` tag with the CSS custom properties, and sets `data-theme` based on the resolved preference. New verticals override these values in their own row — no code change required.
